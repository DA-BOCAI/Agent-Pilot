# AgentCopilot

面向 IM 赛道的 Agent 编排后端。当前主流程是：

`飞书 IM 触发需求 -> Agent 规划任务 -> confirm1 确认需求理解 -> 生成结构化预览 -> 工作台精修/多端同步 -> confirm2 确认正式创建 -> 飞书文档/PPT 交付`

## 核心能力

- 飞书 IM 单聊/群聊入口，支持任务触发、上下文捕捉和确认前补充信息。
- LLM Planner 将自然语言需求拆解为文档、PPT、通知和交付步骤。
- 复杂 PPT/汇报/宣讲类需求会强化为 Doc -> PPT 组合编排：先沉淀文档，再基于文档生成演示稿。
- 文档/PPT 双确认链路：confirm1 核对需求理解，confirm2 核对预览并正式创建。
- 工作台快照与 SSE：首屏用快照渲染，后续用 SSE 同步多端状态。
- 工作台预览精修：支持结构化编辑和自然语言精修，精修结果会作为正式飞书产物的数据源。
- 飞书卡片联动：任务计划、确认、完成卡片与工作台状态互相同步。
