# 项目结构与分层说明

## 项目定位

本项目是 IM 赛道 Agent-Pilot 的后端编排服务，负责把飞书 IM 中的自然语言需求转为任务，并串联规划、确认、预览、精修、飞书产物创建和交付。

## 技术栈

- Java 17
- Spring Boot 3
- Spring Web / SSE
- Spring Data Redis，任务存储优先 Redis，异常时回退内存
- Springdoc OpenAPI
- JUnit 5 + MockMvc

## 当前主流程

1. `LarkImEventListener` 捕捉飞书 IM 消息，识别新任务或补充信息。
2. `AgentTaskService` 创建任务，`AgentRunner` 调用 `LlmPlanner` 规划并推进。
3. 对复杂 PPT/汇报/宣讲类需求，规划器会强制补齐 `C_DOC -> D_SLIDES`：先沉淀文档，再基于文档生成演示稿。
4. 进入 confirm1 时，飞书卡片和工作台用于确认 Agent 是否理解需求。
5. confirm1 通过后，`StepPreviewGenerator` 生成文档/PPT 结构化预览；PPT 预览会优先读取前序文档预览内容作为依据。
6. 工作台通过快照和 SSE 展示预览，并调用工作台专用接口保存/自然语言精修。
7. confirm2 通过后，`LarkToolExecutor` 使用最新 `previewData` 正式创建飞书产物。
8. 完成卡片和工作台 outputs 展示交付链接。
