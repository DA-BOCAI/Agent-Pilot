# 项目结构与分层说明

## 1. 项目定位
本项目是飞书赛题的后端编排服务，负责把自然语言请求转为可执行任务，并驱动步骤执行与确认。

## 2. 当前技术栈
- Java 17
- Spring Boot 3
- Spring Web
- Spring Data Redis（任务存储优先 Redis）
- Lombok
- Springdoc OpenAPI（Swagger）
- JUnit 5 + MockMvc

## 4. 分层职责
- api 控制层：接收请求、参数校验、返回响应。
- service 业务层：维护任务状态机、编排执行流程。
- store 存储层：任务数据读写，优先 Redis，失败自动回退内存。
- planner 规划层：把输入拆为步骤（当前 Mock，后续换 LLM）。
- tool 工具层：执行外部动作（当前 Mock，后续接 MCP CLI）。
- mapper 映射层：领域对象与接口对象转换。

## 5. 为什么去掉 repository
本次按你的要求去掉 repository 包，改为直接 Redis 存储组件 `TaskStore`：
- 减少中间抽象层，便于快速落地。
- 保持后续切 MySQL 的扩展点（可在 store 层做双写或切换）。
