## 主线联调顺序

当前演示主线以飞书 IM 和工作台为核心：

1. 飞书 IM 单聊或群聊 @ 机器人发起需求。
2. Agent 创建任务并自动推进到 confirm1。
3. 用户在飞书卡片或工作台确认需求理解。
4. Agent 生成结构化预览并进入 confirm2。
5. 前端工作台通过快照接口首屏渲染，通过 SSE 接收更新。
6. 用户在工作台结构化编辑或自然语言精修预览。
7. 用户确认 confirm2，Agent 使用最新 previewData 正式创建飞书文档/PPT。
8. 工作台 outputs 和飞书完成卡片展示正式交付链接。

## 工作台核心接口

```http
GET /api/v1/tasks/{taskId}/workspace
GET /api/v1/tasks/{taskId}/workspace/stream
POST /api/v1/tasks/{taskId}/workspace/confirm
POST /api/v1/tasks/{taskId}/workspace/cancel
PUT /api/v1/tasks/{taskId}/workspace/steps/{stepId}/preview
POST /api/v1/tasks/{taskId}/workspace/steps/{stepId}/preview/refine
```

工作台保存/精修接口都会返回完整 `TaskWorkspaceView`，前端可以直接替换当前页面状态。

## PowerShell 调试示例

```powershell
$base = "http://localhost:8080/api/v1/tasks"

$task = Invoke-RestMethod -Method Post -Uri $base -ContentType "application/json" -Body '{"inputText":"请生成新品发布会汇报PPT","requestId":"req-2001","source":"im_text","userId":"u-01"}'
$taskId = $task.taskId

Invoke-RestMethod -Method Post -Uri "$base/$taskId/run"
$workspace = Invoke-RestMethod -Method Get -Uri "$base/$taskId/workspace"

Invoke-RestMethod -Method Post -Uri "$base/$taskId/workspace/confirm" -ContentType "application/json" -Body (@{
  stepId = $workspace.confirmation.stepId
  approved = $true
  source = "workspace"
  clientId = "desktop-web"
} | ConvertTo-Json)
```
