# 最终接口联调文档

本文档为前端最终联调口径。工作台前端以 `TaskWorkspaceView` 为唯一页面状态源：进入页面先拉快照，再订阅 SSE；保存、精修、确认、取消接口均返回完整 `TaskWorkspaceView`，前端成功后直接整体替换本地状态。

## 统一约定

- Base path：`/api/v1`
- `taskId`：任务 ID。
- `stepId`：步骤 ID，常见值为 `C_DOC`、`D_SLIDES`、`SEND_IM`、`F_DELIVER`。
- `source`：建议传 `workspace`。
- `clientId`：建议传 `desktop-web`、`mobile-web`，用于多端同步与时间轴来源展示。
- 所有工作台接口成功后都返回完整 `TaskWorkspaceView`。
- 前端不要自行合并 diff，直接用响应体或 SSE 的最新 `TaskWorkspaceView` 覆盖当前页面状态。

## 1. 获取工作台快照

```http
GET /api/v1/tasks/{taskId}/workspace
```

用途：进入工作台首屏时获取任务、步骤、进度、确认态、预览、交付物、时间轴等完整数据。

请求参数：

| 参数 | 位置 | 必填 | 说明 |
|---|---|---:|---|
| `taskId` | path | 是 | 任务 ID |

返回：`TaskWorkspaceView`

关键字段：

| 字段 | 说明 |
|---|---|
| `taskId` | 任务 ID |
| `title` / `inputSummary` | 页面标题和需求摘要 |
| `status` / `displayStatus` | 后端任务状态和展示文案 |
| `progress.percent` | 后端权威进度百分比 |
| `progress.phase` | 当前进度阶段 |
| `steps[]` | 工作台左侧步骤列表 |
| `confirmation` | 当前是否需要确认，以及确认 stepId/phase |
| `preview` | 当前主预览 |
| `previews[]` | 多产物预览，Doc 和 PPT 都会保留 |
| `outputs[]` | 正式交付物链接 |
| `timeline[]` | 时间轴事件 |
| `diagnostics` | 调试辅助信息，含后端权威进度和预览 warning |

## 2. 订阅工作台实时更新

```http
GET /api/v1/tasks/{taskId}/workspace/stream
```

用途：通过 SSE 接收任务状态、预览、确认态、交付物、时间轴更新。

请求参数：

| 参数 | 位置 | 必填 | 说明 |
|---|---|---:|---|
| `taskId` | path | 是 | 任务 ID |

返回：SSE event，数据体为完整 `TaskWorkspaceView`。

前端处理：

- 建连后仍以 `GET /workspace` 首屏快照为准。
- 收到 SSE 后使用完整对象覆盖本地状态。
- 可用 `preview.data.revision` 判断预览版本是否更新。
- 断线后重连，并重新拉一次 `GET /workspace` 兜底。

## 3. 工作台确认

```http
POST /api/v1/tasks/{taskId}/workspace/confirm
Content-Type: application/json
```

用途：工作台确认当前等待确认的步骤。confirm1 用于确认 Agent 理解，confirm2 用于确认预览并正式创建飞书产物。

请求体：

```json
{
  "stepId": "D_SLIDES",
  "approved": true,
  "comment": "确认预览，可以正式创建",
  "source": "workspace",
  "clientId": "desktop-web"
}
```

参数：

| 字段 | 必填 | 说明 |
|---|---:|---|
| `stepId` | 是 | 当前 `confirmation.stepId` |
| `approved` | 是 | `true` 表示确认通过 |
| `comment` | 否 | 用户备注 |
| `source` | 否 | 建议 `workspace` |
| `clientId` | 否 | 客户端标识 |

返回：完整 `TaskWorkspaceView`。

前端注意：

- 只调用这一个确认接口，不要调用 legacy `/confirm` 或 `/confirm-and-run`。
- confirm2 只应由用户明确点击确认触发。
- 如果接口返回后状态仍在等待，请以返回体和 SSE 为准，不要本地强行推进。

## 4. 工作台取消

```http
POST /api/v1/tasks/{taskId}/workspace/cancel
Content-Type: application/json
```

用途：取消当前确认步骤或任务，并同步 IM 卡片。

请求体：

```json
{
  "stepId": "D_SLIDES",
  "comment": "用户取消本次生成",
  "source": "workspace",
  "clientId": "desktop-web"
}
```

返回：完整 `TaskWorkspaceView`。

## 5. 保存结构化预览

```http
PUT /api/v1/tasks/{taskId}/workspace/steps/{stepId}/preview
Content-Type: application/json
```

用途：前端直接编辑标题、主题、页面结构、章节、slides 后保存。保存后该 `previewData` 会成为 confirm2 正式创建飞书文档/PPT 的数据源。

请求体：

```json
{
  "previewData": {
    "artifactType": "PRESENTATION",
    "title": "校招宣讲材料",
    "theme": "business",
    "pageCount": 12,
    "slides": []
  },
  "source": "workspace",
  "clientId": "desktop-web"
}
```

参数：

| 字段 | 必填 | 说明 |
|---|---:|---|
| `previewData` | 是 | 完整结构化预览数据 |
| `source` | 否 | 建议 `workspace` |
| `clientId` | 否 | 客户端标识 |

返回：完整 `TaskWorkspaceView`。

前端注意：

- 保存成功后直接使用响应体刷新页面。
- 后端会维护 `revision`、`updatedAt`、`lastEditType` 等字段。
- PPT 的 `slides[]`、`speakerNotes`、`rehearsalTips`、`reviewChecklist` 等字段应原样保留并回传。

## 6. 自然语言精修预览

```http
POST /api/v1/tasks/{taskId}/workspace/steps/{stepId}/preview/refine
Content-Type: application/json
```

用途：用户在工作台输入自然语言，让 Agent 修改当前 Doc/PPT 预览。例如“把第 5 页标题改得更吸引人”“整体更正式”“减少 bullet 数量”。

请求体：

```json
{
  "instruction": "把第 5 页标题改得更吸引人，并减少 bullet 数量",
  "source": "workspace",
  "clientId": "desktop-web"
}
```

参数：

| 字段 | 必填 | 说明 |
|---|---:|---|
| `instruction` | 是 | 自然语言精修指令 |
| `source` | 否 | 建议 `workspace` |
| `clientId` | 否 | 客户端标识 |

返回：完整 `TaskWorkspaceView`。

前端注意：

- 调用前应确保当前步骤已经有 `preview.data`。
- 请求期间禁用重复提交。
- 成功后直接使用响应体刷新页面。
- 后端会记录 `lastEditType=natural_language_refine`、`lastRefineInstruction`、`lastRefinedAt`。

## 7. 精确修改预览文字

```http
PATCH /api/v1/tasks/{taskId}/workspace/steps/{stepId}/preview/text
Content-Type: application/json
```

用途：前端支持“点选某段文字后直接改文字”时调用。适用于 PPT 和 Doc 预览，优先使用 `editableTextId` 定位。

请求体优先写法：

```json
{
  "editableTextId": "slide:s5:title",
  "value": "我们和广工有 5 年深度合作，是你身边的可靠本地雇主",
  "source": "workspace",
  "clientId": "desktop-web"
}
```

请求体兼容写法：

```json
{
  "slideId": "slide-5",
  "slideNo": 5,
  "target": "title",
  "value": "新的页面标题",
  "source": "workspace",
  "clientId": "desktop-web"
}
```

可选定位字段：

| 字段 | 说明 |
|---|---|
| `editableTextId` | 后端生成的稳定文本定位，优先使用 |
| `slideId` | PPT slide 稳定 ID |
| `slideNo` | 1-based 页码 |
| `target` | `title`、`bodyMarkdown`、`speakerNotes`、`bullet`、`blockText`、`blockItem`、`tableCell` |
| `bulletIndex` | bullet 下标 |
| `blockIndex` | block 下标 |
| `itemIndex` | block item 下标 |
| `rowIndex` | 表格行下标 |
| `cellIndex` | 表格列下标 |
| `value` | 新文本 |

返回：完整 `TaskWorkspaceView`。

前端注意：

- 首选 `editableTextId + value`。
- 如果没有 `editableTextId`，再用 `slideId/slideNo + target + index` 定位。
- 修改成功后直接整体刷新状态。

## 8. 查询数字卡片视图

```http
GET /api/v1/tasks/{taskId}/cards
```

用途：如前端需要复用飞书数字卡片数据，可读取该接口。

返回：`TaskCardView`，包含：

| 字段 | 说明 |
|---|---|
| `steps[]` | 卡片步骤进度 |
| `confirm` | 当前确认卡片数据 |
| `links[]` | 交付链接 |

## 9. 查询任务事件

```http
GET /api/v1/tasks/{taskId}/events
```

用途：调试和时间轴排查。

返回：`TaskEvent[]`。

## 10. 调试接口

以下接口仅用于本地调试或兼容，不作为前端工作台正式联调入口：

```http
POST /api/v1/tasks
POST /api/v1/tasks/{taskId}/plan
POST /api/v1/tasks/{taskId}/execute
POST /api/v1/tasks/{taskId}/run
POST /api/v1/tasks/{taskId}/steps/{stepId}/preview
GET  /api/v1/tasks/{taskId}
```

说明：

- 正式 IM 演示主线通常由飞书 IM 监听器创建任务并自动推进。
- 手动调试时可用这些接口造任务、规划、执行，但不要混入工作台确认和精修流程。

## 11. 禁用 Legacy 接口

前端不要再调用以下接口：

```http
POST /api/v1/tasks/{taskId}/confirm
POST /api/v1/tasks/{taskId}/confirm-and-run
PUT  /api/v1/tasks/{taskId}/steps/{stepId}/preview
POST /api/v1/tasks/{taskId}/steps/{stepId}/preview/refine
```

替代关系：

| Legacy 接口 | 正式替代接口 |
|---|---|
| `POST /confirm` | `POST /workspace/confirm` |
| `POST /confirm-and-run` | `POST /workspace/confirm` |
| `PUT /steps/{stepId}/preview` | `PUT /workspace/steps/{stepId}/preview` |
| `POST /steps/{stepId}/preview/refine` | `POST /workspace/steps/{stepId}/preview/refine` |

## 12. 推荐前端状态流

1. 进入工作台：调用 `GET /workspace`。
2. 建立 SSE：调用 `GET /workspace/stream`。
3. 渲染：
   - `confirmation.phase=confirm1`：展示需求理解确认。
   - `confirmation.phase=confirm2` 且 `preview.available=true`：展示预览精修。
   - `outputs[]` 非空：展示正式交付物。
4. 用户编辑结构化内容：调用 `PUT /workspace/steps/{stepId}/preview`。
5. 用户自然语言精修：调用 `POST /workspace/steps/{stepId}/preview/refine`。
6. 用户点选文字精确修改：调用 `PATCH /workspace/steps/{stepId}/preview/text`。
7. 用户确认：调用 `POST /workspace/confirm`。
8. 用户取消：调用 `POST /workspace/cancel`。
9. 任一接口成功后：直接用响应体替换本地状态。
10. 任一接口失败：保留本地编辑态，提示用户重试，并以 SSE/快照重新校准。
