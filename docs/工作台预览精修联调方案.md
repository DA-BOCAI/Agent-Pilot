# 工作台预览精修联调方案

## 目标

让飞书端到端演示中的“工作台预览精修”稳定可演示：

1. 用户在工作台修改标题、主题、页面/章节结构后，页面立即显示最新预览。
2. 用户输入自然语言精修指令后，页面直接拿到精修后的完整工作台快照。
3. 桌面端和移动端通过 SSE 保持一致。
4. 最终点击 confirm2 后，后端使用精修后的 `previewData` 正式创建飞书文档或 PPT。

## 后端改动摘要

本轮后端已补齐：

- 工作台专用保存接口：返回完整 `TaskWorkspaceView`。
- 工作台专用自然语言精修接口：返回完整 `TaskWorkspaceView`。
- `adjustments.actions[].endpoint` 已切换到工作台专用接口。
- `previewData` 自动增加 `revision`、`lastEditType`、`updatedAt`、`lastRefineInstruction`、`lastRefinedAt`。
- 对 `C_DOC -> D_SLIDES` 组合编排，PPT 预览会自动带上 `composition.mode=doc_to_ppt` 和来源文档版本信息。
- 时间轴事件会记录 `source` 和 `clientId`，用于区分桌面端/移动端。
- 每次保存/精修都会触发 SSE 广播最新工作台快照。

## 接口 1：保存结构化预览

用于前端直接编辑标题、主题、页面结构、文档章节等。

```http
PUT /api/v1/tasks/{taskId}/workspace/steps/{stepId}/preview
Content-Type: application/json
```

请求体：

```json
{
  "previewData": {
    "artifactType": "PRESENTATION",
    "title": "校招宣讲方案",
    "theme": "business",
    "pageCount": 3,
    "estimatedDurationMinutes": 3,
    "rehearsalTips": [
      "建议总时长控制在约 3 分钟，每页保留一个明确结论。"
    ],
    "reviewChecklist": [
      "确认封面标题、汇报对象和场景一致。"
    ],
    "slides": []
  },
  "source": "workspace",
  "clientId": "desktop-web"
}
```

返回：完整 `TaskWorkspaceView`。

前端处理：

- 请求成功后直接用响应体替换当前页面状态。
- 不需要再额外请求 `GET /workspace`。
- 失败时保留本地编辑内容，并提示用户重试。

## 接口 2：自然语言精修

用于“把风格改商务一点”“删掉第三页”“标题改成 xxx”等指令。

```http
POST /api/v1/tasks/{taskId}/workspace/steps/{stepId}/preview/refine
Content-Type: application/json
```

请求体：

```json
{
  "instruction": "把整体风格改得更适合销售团队，标题改为校招增长作战方案",
  "source": "workspace",
  "clientId": "desktop-web"
}
```

返回：完整 `TaskWorkspaceView`。

后端行为：

- 优先调用大模型精修结构化 `previewData`。
- 大模型不可用或输出无效时，降级为确定性规则精修，至少支持标题和主题修改。
- 精修后的 `previewData` 会成为 confirm2 正式创建飞书产物的唯一数据源。

## 前端推荐状态流

### 初始进入 confirm2

1. 读取快照 `GET /api/v1/tasks/{taskId}/workspace`。
2. 如果 `confirmation.phase=confirm2` 且 `preview.available=true`，渲染预览编辑器。
3. 使用 `adjustments.actions` 渲染可编辑能力。

### 用户结构化编辑

1. 用户修改标题/主题/页面。
2. 前端本地进入 `saving_preview` 状态。
3. 调用 `PUT /workspace/steps/{stepId}/preview`。
4. 用响应体替换页面状态。
5. 退出 `saving_preview`。

### 用户自然语言精修

1. 用户输入指令。
2. 前端进入 `refining_preview` 状态，禁用重复提交。
3. 调用 `POST /workspace/steps/{stepId}/preview/refine`。
4. 用响应体替换页面状态。
5. 清空输入框或保留最近指令展示。
6. 退出 `refining_preview`。

### 多端同步

1. 当前端已经建立 SSE，另一端保存或精修后会收到 `workspace` 事件。
2. SSE 数据也是完整 `TaskWorkspaceView`。
3. 前端可比较 `preview.data.revision`，如果 SSE 的 revision 更新，则替换页面状态。

## 关键字段

精修后 `preview.data` 示例：

```json
{
  "artifactType": "PRESENTATION",
  "title": "校招增长作战方案",
  "theme": "business",
  "pageCount": 3,
  "estimatedDurationMinutes": 3,
  "slides": [],
  "rehearsalTips": [
    "建议总时长控制在约 3 分钟，每页保留一个明确结论。"
  ],
  "reviewChecklist": [
    "确认封面标题、汇报对象和场景一致。",
    "核对关键数字、口径和来源，避免指标页空泛。"
  ],
  "composition": {
    "mode": "doc_to_ppt",
    "source": {
      "stepId": "C_DOC",
      "type": "doc",
      "artifactType": "docs-preview",
      "title": "校招方案文档",
      "revision": 1,
      "updatedAt": "2026-05-03T09:58:00Z"
    }
  },
  "revision": 2,
  "lastEditType": "natural_language_refine",
  "updatedAt": "2026-05-03T10:00:00Z",
  "lastRefineInstruction": "把整体风格改得更适合销售团队，标题改为校招增长作战方案",
  "lastRefinedAt": "2026-05-03T10:00:00Z"
}
```

字段说明：

| 字段 | 含义 |
|---|---|
| `revision` | 预览版本号，生成预览为 1，之后每次保存/精修 +1 |
| `lastEditType` | `generated`、`manual_update`、`natural_language_refine` |
| `updatedAt` | 最近一次预览数据更新时间 |
| `lastRefineInstruction` | 最近一次自然语言精修指令 |
| `lastRefinedAt` | 最近一次自然语言精修时间 |
| `composition.mode` | `doc_to_ppt` 表示当前 PPT 预览基于前序文档生成 |
| `composition.source` | 来源文档预览信息，包含 `stepId`、标题和版本号 |
| `estimatedDurationMinutes` | PPT 预计讲解时长，前端可用于排练区展示 |
| `rehearsalTips` | 整体排练建议，适合在 PPT 预览侧栏或讲稿区展示 |
| `reviewChecklist` | 交付前检查清单，用于提醒用户核对封面、数据、时间线、下一步 |
| `slides[].speakerNotes` | 单页讲稿提示，用于每页备注/排练面板 |
| `slides[].layout` | 后端推荐版式，如 `cover`、`content`、`two_column`、`metric_cards`、`timeline`、`comparison_table`、`closing` |

前端建议：

- 用 `revision` 判断是否收到新版本。
- 用 `lastEditType` 显示“已保存/已精修/预览已生成”。
- 用 `lastRefineInstruction` 在精修输入区下方展示最近一次指令。
- 当 `preview.type=slides` 且 `preview.data.composition.mode=doc_to_ppt` 时，可展示“基于《source.title》v{source.revision} 生成”。

## adjustments 动作渲染

工作台快照中会返回：

```json
{
  "adjustments": {
    "available": true,
    "stepId": "D_SLIDES",
    "phase": "confirm2",
    "actions": [
      {
        "key": "theme",
        "label": "切换主题",
        "type": "select",
        "target": "preview.data.theme",
        "options": ["business", "tech", "campaign", "minimal"],
        "currentValue": "business",
        "endpoint": "/api/v1/tasks/task-1/workspace/steps/D_SLIDES/preview",
        "method": "PUT"
      },
      {
        "key": "naturalLanguageRefine",
        "label": "自然语言精修",
        "type": "instruction",
        "target": "preview.data",
        "options": [],
        "endpoint": "/api/v1/tasks/task-1/workspace/steps/D_SLIDES/preview/refine",
        "method": "POST"
      }
    ]
  }
}
```

前端不要硬编码旧接口，优先使用 action 里的 `endpoint` 和 `method`。

## 时间轴联调

保存结构化预览后会新增：

```json
{
  "type": "STEP_PREVIEW_UPDATED",
  "title": "预览已更新",
  "source": "workspace",
  "sourceDisplay": "工作台",
  "metadata": {
    "stepId": "D_SLIDES",
    "artifactType": "slides-preview",
    "source": "workspace",
    "clientId": "desktop-web",
    "composition": "doc_to_ppt",
    "sourceStepId": "C_DOC",
```


## 验收用例

### 用例 1：修改标题后预览立即更新

1. 打开 confirm2 工作台。
2. 修改标题为“校招增长作战方案”。
3. 调用结构化保存接口。
4. 页面使用响应体更新。

验收：

- 标题立即变更。
- `preview.data.revision` 增加。
- 时间轴出现“预览已更新”。

### 用例 2：自然语言精修后不二次拉取

1. 输入“改成商务稳重风格，标题改为校招增长作战方案”。
2. 调用自然语言精修接口。
3. 页面直接使用响应体更新。

验收：

- 主题变为 `business`。
- 标题变为新标题。
- `preview.data.lastEditType=natural_language_refine`。
- `preview.data.lastRefineInstruction` 有值。
- 时间轴出现“预览已精修”。

### 用例 3：移动端精修，桌面端同步

1. 桌面端打开工作台并建立 SSE。
2. 移动端执行自然语言精修，`clientId=mobile-web`。
3. 桌面端收到 SSE `workspace` 事件。

验收：

- 桌面端预览内容更新。
- 时间轴来源显示“工作台”。
- metadata 中 `clientId=mobile-web`。

### 用例 4：精修后正式创建飞书 PPT

1. 在 confirm2 工作台完成精修。
2. 点击确认。
3. 后端正式创建飞书 PPT。

验收：

- 飞书 PPT 使用精修后的标题、主题和 slides。
- 完成卡片和工作台 outputs 中出现正式飞书链接。

## 错误处理建议

| 场景 | 后端响应 | 前端处理 |
|---|---|---|
| 未生成预览就精修 | 400：请先生成预览，再进行精修 | 回到生成预览/等待态 |
| 指令为空 | 400：精修指令不能为空 | 输入框提示 |
| previewData 类型不匹配 | 400：artifactType 必须是指定类型 | 保留本地编辑，提示刷新 |
| 大模型不可用 | 后端规则降级，接口仍返回 200 | 正常展示，必要时提示“已做基础精修” |
